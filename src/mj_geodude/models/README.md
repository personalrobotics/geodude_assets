Geodude MuJoCo model files.

## Contents

- [vention](./vention/): MuJoCo model of the vention base
- [universal_robots_ur5e](./universal_robots_ur5e/): MuJoCo model of UR5e
- [robotiq_2f140](./robotiq_2f140/): MuJoCo model of Robotiq 2F-140 Gripper
- [abh_left_small](./abh_left_small/): MuJoCo model of the Psyonic Ability Hand (spec: left small)
- [abh_right_small](./abh_right_small/): MuJoCo model of the Psyonic Ability Hand (spec: right small)
- [robot_assembly.py](./robot_assembly.py): Creates a MuJoCo model of two UR5es attached to the vention base.
This can be used to either generate the `mujoco.MjModel` object for use in python code, or to save the MuJoCo model to a directory for inspection/future use.
- [geodude](./geodude/): The MuJoCo model that was generated by `robot_assembly.py`.
The contents of this directory should be treated as "read only" and not modified directly.
If the geodude model needs to be updated, this should be done by modifying the standalone models and then re-running `robot_assembly.py`.

The geodude model is programmatically generated with `robot_assembly.py`.
This allows for individual components (vention, UR5e, hands, etc) to be inspected independently if needed.
This also allows for easy interchanging of things like hands on the robot.

```
python robot_assembly.py -h
```

For more information about PyMJCF and attaching bodies to MuJoCo models programmatically, see:
- https://github.com/google-deepmind/mujoco_menagerie/blob/main/FAQ.md
- https://github.com/google-deepmind/dm_control/tree/main/dm_control/mjcf
